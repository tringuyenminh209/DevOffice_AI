Arrow icon: 4×20px shaft + 16×4px arrowhead (geometric, no emoji). - Footer note 10px text-muted: QR code mention. --- Technical Execution - Framework: React + Tailwind CSS. Use dark: variants for all colors. - Pixel Canvas: Phaser.js element embedded inside a React component, rendering at exactly 1280×720px internal resolution scaled to fit. - Whitespace: 80px minimum vertical gap between major sections on the landing page. 16–24px gap between cards within sections. - No gradients except background ambient glow orbs (radial, very low opacity). All UI surfaces use flat solid fills from the token palette. - No rounded corners above 20px anywhere in the UI. - Hover states: Subtle 1px border changes from border to accent color. Card backgrounds shift one step up the surface stack. 150ms ease transitions only — no bouncy spring animations. - Agent sprites on the canvas must use z-index based on y-coordinate (Y-sorting) — agents lower on screen render in front of agents higher on screen, creating 2.5D depth. Agents with approval.required state always render at z-index: 1000. - Smart Bubbles in Manager View show translated Vietnamese business language, never raw technical logs. In Dev View, raw monospaced English log syntax is used. - Color meaning is absolute and must be consistent across all screens: emerald=success/idle, amber=waiting/cost, crimson=error/high-risk, accent violet=active/brand, sapphire=informational.
